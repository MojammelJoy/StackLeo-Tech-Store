# Changelog

## 1. Purpose

This document tracks significant updates made to the documentation within the `00_Project_Overview` folder for the **StackLeo Tech Store** project. It serves as the historical record of documentation changes throughout the project lifecycle, providing traceability for how the project's foundational understanding has evolved over time.

This changelog records changes to documentation only. It does not track code changes, technical implementation history, or system-level release notes, which are addressed in dedicated technical documentation elsewhere in the repository.

## 2. Versioning Policy

Documentation within `00_Project_Overview` follows [Semantic Versioning](https://semver.org/) (SemVer) principles, using a `MAJOR.MINOR.PATCH` format:

| Segment | When It Changes |
|---|---|
| MAJOR | A fundamental change in project direction, scope, or business model that redefines how the documentation should be understood. |
| MINOR | The addition of new sections, new documents, or meaningful expansions to existing content that do not change prior meaning. |
| PATCH | Small corrections, clarifications, wording improvements, or formatting updates that do not change meaning or scope. |

Each document in this folder maintains its own version number, reflected in that document's `Document Information` section. This changelog records both individual document updates and folder-wide milestones.

## 4. Upcoming Documentation

The following documentation activity is anticipated as the project progresses beyond its current Planning & Documentation stage:

- Updates to `project-scope.md`, `project-goals.md`, and `project-roadmap.md` as the project advances through the phases defined in the roadmap.

## 5. Maintenance Guidelines

- Every meaningful change to a document within `00_Project_Overview` must be recorded in this changelog.
- Changelog entries should reference the version, date, type of change, and a concise summary of what changed and why.
- The version number of an updated document must be incremented according to the versioning policy defined in this document before the change is recorded here.
- Entries should be added in reverse chronological order, with the most recent version listed first.
- This changelog should be reviewed for accuracy whenever any document in this folder is added, updated, or removed.
