# Customer FAQ & Knowledge Base

## Document Purpose

This document is the official Customer Knowledge Base for **StackLeo Tech Store**. It serves as the primary FAQ reference for customers, customer support, the sales team, operations, marketing, the call center, and — in the future — the AI chatbot and self-service Help Center.

Every answer in this document is grounded in StackLeo's official policies. Where relevant, each entry references the related policy area and the underlying document that governs it, so that support staff, AI systems, and customers can consult the authoritative source for full detail. This document does not describe implementation approach, technology choices, API design, or database structure.

## How to Use This Document

- Questions are organized into 35 categories, listed in the index below, covering the customer journey from account creation through post-purchase support.
- Each question has a unique identifier in the format `FAQ-XXX` for easy reference by support staff and future AI chatbot systems.
- Each answer includes a **Related Policy**, **Related Document**, and, where useful, a **Note** pointing to further detail or an important caveat.
- Search Tip: search this document by keyword (e.g., "refund," "warranty," "EMI") or by category name to quickly locate relevant answers.
- Search Tip: if a question spans multiple categories (e.g., "Can I return a product I bought with EMI?"), check both related categories, as cross-references are provided.
- If an answer does not fully resolve a question, escalate to human support using the process defined in Section 36.

## 36. Support Escalation

If an FAQ answer does not resolve your issue, follow this escalation path:

1. **Self-Service:** Review the relevant FAQ category and linked policy document above.
2. **Standard Support:** Contact customer support through phone, email, or live chat with your order number and issue description.
3. **Escalated Support:** If unresolved within the expected response time, request escalation to a senior support representative.
4. **Formal Complaint:** For unresolved disputes, a formal complaint may be submitted for review by the operations team, consistent with the governance process defined in `business-rules.md` (Section 20).
